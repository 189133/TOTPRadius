# TOTP Radius认证服务

这是一个基于Python + Flask + SQLite + TOTP的Radius认证服务，提供Web界面的用户管理功能和TOTP双因素认证。

## 功能特性

- ✅ 用户管理：添加、删除、启用/禁用用户
- ✅ TOTP双因素认证：支持Google Authenticator等认证器
- ✅ Web管理界面：响应式设计，操作简单直观
- ✅ Radius认证服务：支持标准Radius协议
- ✅ 二维码生成：方便用户快速添加TOTP密钥到认证器
- ✅ HTTP认证：保护管理界面
- ✅ 配置管理：通过Web界面管理调试模式和共享密钥
- ✅ 详细日志：记录认证过程和结果
- ✅ 支持PAP和CHAP认证
- ✅ 支持Status-Server请求
- ✅ 支持Accounting功能
- ✅ **宽容认证**：支持断网重连自动认证，提高用户体验
  - 12小时重连宽容：相同用户、协议、IP的认证失败时，使用历史成功密码
  - 10分钟最近认证宽容：相同用户、协议、服务器IP的认证失败时，使用历史成功密码
- ✅ **认证日志**：记录详细的认证信息，包括用户名、时间、协议、IP、密码、结果等
- ✅ **安全黑名单**：防止暴力破解，连续失败后临时锁定用户

## 技术栈

- **后端**：Python 3.9+, Flask 2.0.1
- **数据库**：SQLite（内置，无需额外安装）
- **认证**：PyOTP 2.6.0（TOTP实现）
- **Radius**：PyRad 2.5.4+（Radius协议实现）
- **前端**：纯HTML/CSS/JavaScript + QRCode.js

## 安装步骤

### 1. 克隆项目

```bash
git clone <repository-url>
cd TOTPRadius
```

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 启动服务

#### 启动Web管理界面和API服务

```bash
cd frontend
python app.py
```

访问：http://localhost:8080

#### 启动Radius服务

```bash
cd backend
python radius_server.py
```

Radius服务默认监听端口：1812（认证）和1813（记账）

## 使用方法

### Web管理界面

1. **访问界面**：在浏览器中打开 http://localhost:8080，系统会提示输入HTTP认证信息：
   - 用户名：`admin`
   - 密码：`admin123`

2. **用户认证**：在"用户认证"标签页输入用户名和TOTP验证码进行认证

3. **用户管理**：在"用户管理"标签页可以：
   - 添加新用户（会自动生成TOTP密钥并显示二维码）
   - 删除用户
   - 启用/禁用用户

4. **配置管理**：在"配置管理"标签页可以：
   - 启用/禁用调试模式（调试模式下，任何密码验证都通过）
   - 修改RADIUS共享密钥
   - 配置黑名单参数（最大失败次数、锁定时间）
   - 启用/禁用宽容认证功能
   - 配置宽容认证的时间窗口
   - 保存配置后，需要重启RADIUS服务器使配置生效

### Radius认证

Radius客户端需要使用以下格式发送认证请求：

- **用户名**：用户的用户名
- **密码**：直接输入TOTP验证码

### 示例

假设当前TOTP验证码是 `123456`，则Radius认证密码应为：

```
123456
```

## 安全说明

- 用户没有密码，只有TOTP密钥
- TOTP密钥以明文存储在SQLite数据库中
- 管理界面受到HTTP认证保护
- 共享密钥存储在数据库中
- 定期备份SQLite数据库文件
- **宽容认证安全**：
  - 宽容认证默认禁用，需要管理员主动启用
  - 宽容认证有时间窗口限制，不会永久有效
  - 宽容认证需要匹配多个条件（用户名、协议、IP等）
  - 宽容认证的成功记录会被标记为特殊代码（1001/1002）
  - 建议根据实际安全需求调整宽容认证的时间窗口
- **黑名单保护**：
  - 连续认证失败达到阈值后，用户会被临时锁定
  - 锁定时间和失败次数阈值可在Web界面配置
  - 黑名单存储在内存中，重启服务后清空

## 配置选项

### Radius服务器配置

- 调试模式：通过Web界面的"配置管理"标签页修改
- 共享密钥：通过Web界面的"配置管理"标签页修改
- 客户端地址：在 `radius_server.py` 中修改
- 监听端口：在 `radius_server.py` 中修改

### 宽容认证配置

宽容认证功能可以在Web界面的"配置管理"标签页中配置：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 启用12小时重连宽容 | 是否启用12小时重连宽容认证 | 禁用 |
| 重连宽容时间 | 重连宽容的时间窗口（秒） | 43200（12小时） |
| 启用10分钟最近认证宽容 | 是否启用10分钟最近认证宽容 | 禁用 |
| 最近认证宽容时间 | 最近认证宽容的时间窗口（秒） | 600（10分钟） |

**宽容认证规则**：

1. **12小时重连宽容**（结果代码：1001）：
   - 条件：相同用户名、协议类型、服务器IP、客户端IP
   - 时间窗口：12小时内（可配置）
   - 使用最近一次成功认证的密码

2. **10分钟最近认证宽容**（结果代码：1002）：
   - 条件：相同用户名、协议类型、服务器IP
   - 时间窗口：10分钟内（可配置）
   - 使用最近一次成功认证的密码

**使用场景**：
- 网络中断后自动重连
- TOTP验证码过期但用户仍在使用旧验证码
- 提高用户体验，减少因网络问题导致的认证失败

### Web服务配置

- 服务端口：在 `frontend/app.py` 中修改
- HTTP认证信息：在 `frontend/app.py` 中修改

## 故障排查

### 常见问题

1. **Radius认证失败**：
   - 检查TOTP验证码是否正确
   - 检查TOTP验证码是否过期
   - 检查用户是否启用
   - 检查共享密钥是否匹配
   - 检查宽容认证是否启用（如果使用旧验证码）

2. **Web界面无法访问**：
   - 检查Flask服务是否启动
   - 检查端口是否被占用
   - 检查HTTP认证信息是否正确

3. **二维码无法扫描**：
   - 检查网络连接（需要加载QRCode.js）
   - 尝试手动输入TOTP密钥

4. **配置修改后不生效**：
   - 检查是否保存了配置
   - 检查是否重启了Radius服务器

5. **宽容认证不生效**：
   - 检查宽容认证功能是否启用
   - 检查配置是否已保存到数据库
   - 检查是否重启了Radius服务器
   - 查看认证日志确认是否符合宽容认证条件

6. **用户被锁定**：
   - 检查是否连续认证失败达到最大次数
   - 等待锁定时间过期
   - 检查黑名单配置是否合理
